G8RTOS Bowl

A real-time American football game built from scratch on a TI Tiva microcontroller using a custom preemptive RTOS, featuring concurrent gameplay, graphics, audio, and input handling.

Project Overview

G8RTOS Bowl was my final project for the course EEL 4745: Microprocessor Applications 2 (RTOS). It combines a custom real-time operating system with a complete, playable American football game. The project runs on a TI Tiva™ TM4C123 LaunchPad + Multimod and demonstrates real-world RTOS concepts such as preemptive scheduling, semaphores, FIFOs, periodic and aperiodic events, and dynamic threads, all under real timing and hardware constraints.

The game includes animated players, route-based AI movement, real-time collision detection, joystick-controlled gameplay, sound effects, and a full football state machine (downs, drives, scoring, kickoffs, turnovers). Watch my demo on YouTube !

Hardware & Platform

  • MCU: TI TM4C123GH6PM (ARM Cortex-M4F)
  • Display: ST7789 LCD (SPI)
  • Inputs: Analog joystick + push buttons (GPIO + I2C expander)
  • Audio: UART-based sound trigger system
  • Peripherals Used: SPI, UART, ADC, GPIO, I2C, timers, interrupts

System Architecture

Custom RTOS Built from Scratch (G8RTOS)

  • Preemptive, priority-based scheduler
  • Dynamic thread creation and termination
  • Blocking, sleeping, and yielding threads
  • Spin-lock semaphores for shared resources (SPI, UART, I/O)
  • FIFO-based inter-process communication
  • Periodic, aperiodic (interrupt-driven), and background threads

Thread Model

Background / System Threads

  • Idle thread
  • Periodic screen redraw thread

Gameplay Threads (Dynamic)

  • QB, WR, DB behavior threads
  • Ball physics thread
  • Passing, tackling, touchdown, and incomplete-pass handlers
  • Game state transition threads (kickoff, play select, fourth down, end game)

Interrupt-Driven Threads

  • GPIO button interrupts
  • Joystick press interrupts

Threads are created and killed dynamically to match gameplay state, minimizing CPU idle time and keeping the system responsive.

Gameplay Features

  • Route-based receiver AI (slant, go, curl, in, out, post, corner)
  • Joystick-controlled quarterback and ball carrier
  • Real-time collision detection (DB vs WR/QB)
  • Football logic: downs, line of scrimmage, end zones, touchdowns, field goals, turnovers, drives
  • Animated sprites with flicker-free rendering
  • Audio cues triggered by game events (touchdown, tackle, incomplete, win/loss)
  • Custom bitmap sprites for players, UI, and menus

Concurrency & Synchronization

  • Semaphores protect SPI, UART, joystick, and button access
  • FIFOs send data between threads
  • Periodic events handle rendering every 100 ms
  • Aperiodic events respond instantly to hardware interrupts

Tech Stack

  • Language: C
  • RTOS: Custom G8RTOS
  • Toolchain: TI Code Composer Studio
  • Drivers: Custom Multimod peripheral drivers
