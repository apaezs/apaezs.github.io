FPGA-Accelerated 1-D Time-Domain Convolution

A high-throughput FPGA convolution accelerator with a custom DRAM DMA interface, clock-domain crossing, and a fully pipelined multiply-add datapath.

Project Overview

This project implements a hardware-accelerated 1-D time-domain convolution engine on the Xilinx Zedboard, targeting large performance gains over a software CPU implementation. The design exploits massive parallelism and pipelining to stream data directly from DRAM, compute convolution in hardware, and write results back to memory.

The system is split into two major components:

  • A custom DRAM read DMA engine with safe clock-domain crossing
  • A fully unrolled, pipelined convolution datapath supporting kernels up to 128 elements

The final design achieves up to ~16× speedup on large workloads.

Platform & Tools

  • Board: Xilinx Zedboard (Zynq-7000)
  • Clocks:
    • DRAM / DMA domain: 100 MHz
    • User / convolution domain: 70 MHz
  • Languages: VHDL (primary), C++
  • Tools: Vivado, Questa

System Architecture

High-Level Dataflow

DRAM → DMA Read Engine → Signal Buffer → Convolution Pipeline → Output DRAM

All transfers and control signals safely cross clock domains using dual flop synchronizers, handshake synchronizers, and dual-clock FIFO synchronizers.

Key Features

  • Custom DRAM Interface: Direct memory access with optimized burst transfers
  • Clock-Domain Crossing: Safe synchronization between asynchronous clock domains
  • Fully Pipelined Datapath: Maximum throughput with minimal latency
  • Scalable Architecture: Supports kernels from small to large (up to 128 elements)
  • High Performance: Up to 16× speedup over CPU implementation

Performance Results

The FPGA accelerator demonstrates significant performance improvements:

  • Speedup: Up to ~16× faster than software CPU implementation
  • Throughput: Optimized for large workloads with continuous data streaming
  • Latency: Low-latency processing through pipelined architecture
